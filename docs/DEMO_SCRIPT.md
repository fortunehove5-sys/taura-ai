# Demo Script (suggested, ~4 minutes)

Use this as a starting point for the AI4I bootcamp pitch / adjudicator demo.
All three channel simulators and the web demo run offline against synthetic
sample data (see `docs/DATASET_STATEMENT.md`) -- no external API keys or
network access are required.

## 0. Setup (before you're on stage)

```bash
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
```

Open two terminals: one for the backend, one for the demo commands.

```bash
# terminal 1
uvicorn backend.app:app --reload --port 8000
# then open http://localhost:8000 in a browser
```

## 1. Open with the problem (30s)

"Zimbabwe's economy is now majority informal, and the farmers and traders who
make it up mostly speak Shona or Ndebele, not English, and mostly carry a
feature phone or use USSD, not a banking app."

## 2. Show the WhatsApp-style web demo (90s)

In the browser at `http://localhost:8000`:

1. Type `Mhoro` -- point out the consent prompt appears immediately, in Shona,
   *before* anything else happens.
2. Reply `hongu` -- greeting response appears.
3. Click the quick-reply "Maize price · Mutare" -- point out the **grounding
   chip** under the response (`grounded in: PRICE-MAIZE-MUTARE-2026W27`) and
   say: "This number was not generated by a language model -- it was
   retrieved from a source record and the model only phrased it in Shona."
4. Switch the channel tab to **USSD**, then **WhatsApp**, and repeat a query
   to show the same backend serves all three channels identically.
5. Ask a loan question ("ngicela isikwelede") and point out the response is
   tagged **escalated** -- the AI hands off to a human for anything requiring
   a financial commitment, per the safeguard in Section 4.2 of the written
   proposal.

## 3. Show the CLI voice-call simulator (45s)

```bash
python -m taura.channels.cli_chat
```

Type a mixed-language exchange live to show the multilingual routing working
in a plain terminal (standing in for a phone call), e.g.:

```
mhoro
hongu
inani lomumbila eBulawayo
isikhukhula eChipinge
```

## 4. Show the test suite (30s)

```bash
pytest -q
```

"53 tests cover the consent flow, intent classification in three languages,
entity extraction, retrieval grounding, ASR/TTS backend wiring, and the
escalation rule end to end."

## 5. Close (30s)

"Everything you just saw runs on synthetic sample data and template-based
phrasing so it's fully offline and reviewable today. The architecture is
built so that swapping in a real Shona/Ndebele ASR model, a live AMA/MSD
feed, and a fine-tuned language model are each a single-file change -- see
`docs/ARCHITECTURE.md` for exactly where."

| Question | Where to point |
|---|---|
| "Where's the real AI?" | `docs/ARCHITECTURE.md` -- explain intent classification and RAG are the AI-necessary parts; ASR/TTS/LLM are pluggable and clearly marked as integration points, not hidden |
